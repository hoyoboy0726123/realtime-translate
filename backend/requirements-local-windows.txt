# On-device engine + recording analysis — Windows / Linux.
#
#   pip install -r requirements-local-windows.txt
#
# Speech recognition uses faster-whisper (CTranslate2), the summary LLM uses
# llama-cpp-python (GGUF). Translation (NLLB) runs on CPU via transformers;
# speaker diarization uses sherpa-onnx (ONNX, self-contained, no torch).
#
# As-is this installs a working CPU-only stack. For NVIDIA GPU acceleration
# see the README "Windows / Linux" section — in short:
#   * use CUDA 12.4 + cuDNN 9 (CUDA 11.x and 12.0-12.2 do NOT work);
#   * keep torch CPU-only (a CUDA torch bundles cuDNN 8, which clashes with
#     CTranslate2's cuDNN 9) — NLLB runs on CPU anyway;
#   * install llama-cpp-python from its cu124 wheel index.
#
# transformers is pinned to 4.44.x — newer releases require torch >= 2.4.
# First use downloads the models (Whisper, NLLB ~2.4 GB, Qwen2.5-7B GGUF ~4.7 GB).
# Speaker-diarization models download from sherpa-onnx releases into
# data/diarization/ (see README).

transformers>=4.44,<4.45
torch>=2.2,<2.3
sentencepiece>=0.2
numpy>=1.26,<2
opencc>=1.1
sherpa-onnx>=1.10
faster-whisper>=1.0
llama-cpp-python>=0.3
